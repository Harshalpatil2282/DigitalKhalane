# 🎯 FIX: Server Won't Start - MongoDB Issue

## ❌ Current Problem
```
Error: connect ECONNREFUSED ::1:27017
MongoDB not running on localhost:27017
```

## ✅ Solution: Pick ONE Option Below

---

## 🟢 EASIEST: Use MongoDB Atlas (3 minutes)

### Step 1: Create Free Account
```
Go to: https://account.mongodb.com/account/register
Email: your-email@gmail.com
Password: strong-password
```

### Step 2: Create Cluster
```
After login → Click "CREATE DEPLOYMENT"
Tier: FREE
Region: Choose closest to you
Click "CREATE"
Wait 1-2 minutes...
```

### Step 3: Get Connection String
```
Click "CONNECT"
→ Choose "Drivers"
→ Select "Node.js"
→ Copy the connection string
```

**You'll get something like:**
```
mongodb+srv://<username>:<password>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority
```

### Step 4: Update .env File
**Edit:** `d:\DigitalKhalane\server\.env`

Replace this:
```
MONGO_URI=mongodb://localhost:27017/digitalkhalane
```

With this (your atlas string):
```
MONGO_URI=mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/digitalkhalane?w=majority
```

⚠️ Replace:
- `username` with your MongoDB Atlas username
- `password` with your MongoDB Atlas password  
- `cluster0.xxxxx` with your actual cluster URL

### Step 5: Start Server
```bash
cd d:\DigitalKhalane\server
npm run dev
```

**You should see:**
```
✅ MongoDB Connected Successfully
   Host: cluster0.xxxxx.mongodb.net
   Database: digitalkhalane

🎪  DigitalKhalane API Server Started
🌐  http://localhost:5000
```

✅ **DONE!**


### Error: "connect ECONNREFUSED"
MongoDB not running - pick ONE option above and follow steps

### Error: "Authentication failed"
Check your MONGO_URI in `.env` - credentials might be wrong

### Error: "Cannot connect to MongoDB Atlas"
- Make sure IP is whitelisted: MongoDB Atlas → Security → Network Access
- Add your IP or "Allow Access from Anywhere" for development
